# Module 13 — Live Fleet Map & ETA Sharing

## Overview

Real-time fleet visualization on an interactive map showing all vehicle positions, statuses, and trip routes. Includes public ETA sharing for customers.

---

## Features

### 13.1 Live Fleet Map

- Interactive map (Leaflet + OpenStreetMap tiles)
- All vehicles displayed with status-colored markers
- Click marker to view vehicle details popup
- Filter by status, type, region
- Auto-refresh every 30 seconds
- Cluster markers when zoomed out

### Status Colors

| Status | Color | Icon |
|---|---|---|
| Available | 🟢 Green | Parked vehicle |
| On Trip | 🔵 Blue | Moving vehicle |
| Maintenance | 🟡 Yellow | Wrench |
| Alert | 🔴 Red | Warning triangle |

### 13.2 Trip Route Display

- Active trip routes shown as polylines on map
- Source marker (green flag), destination marker (red flag)
- Current position marker (animated pulse)
- Planned route (dashed line) vs actual path (solid line)

### 13.3 Customer ETA Sharing

**Public Tracking Page** (no authentication required):

```
URL: /track/{trackingToken}
```

Displays:
- Trip status badge (In Transit / Arriving Soon / Delivered)
- Mini map with current position and destination
- ETA countdown
- Delivery progress bar (percentage based on distance)
- Last updated timestamp

Token generated by dispatcher, expires 24h after trip completion.

---

## Data Flow

```
Driver App → GPS Update (lat, lng) → Backend → Store in Redis/DB
                                        ↓
Live Map (polling every 30s) ← GET /api/v1/fleet/positions
Public Track Page (polling every 60s) ← GET /api/v1/trips/track/{token}
```

### Simulated Positions (Hackathon)

For demo purposes, vehicle positions are simulated:
- Interpolate along route geometry based on elapsed time
- Add small random offset for realistic movement
- Update every minute via cron

## API Endpoints

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| GET | `/fleet/positions` | Bearer | All vehicle positions + status |
| GET | `/fleet/positions/:vehicleId` | Bearer | Single vehicle position |
| PUT | `/fleet/positions/:vehicleId` | Bearer | Update vehicle position (driver) |
| GET | `/trips/track/:token` | Public | Public tracking data |
| POST | `/trips/:id/tracking-link` | Bearer | Generate tracking link |

---

## Implementation Notes

- **Leaflet**: react-leaflet for React integration
- **Tiles**: OpenStreetMap (free, no API key)
- **Clustering**: react-leaflet-markercluster for zoom-out views
- **Route rendering**: GeoJSON polylines from OpenRouteService
- **Position storage**: Latest position in DB (or Redis for production scale)
- **Responsive**: Map fills viewport on desktop, stackable with filters on mobile
